Core Cloud Concepts

Cloud Computing

On-demand delivery of IT resources over the Internet with pay-as-you-go pricing. It's a shift from Capital Expenditure (CapEx) to Operational Expenditure (OpEx).

Scalability vs. Elasticity

Scalability: Handling growing workloads. Vertical (scaling up) increases a single server's resources. Horizontal (scaling out) adds more servers.

Elasticity: Automatically scaling resources in response to fluctuating demand.

Availability & Resilience

High Availability (HA): Minimizes downtime.
Fault Tolerance (FT): Continues operating even if components fail.
Disaster Recovery (DR): Restoring operations after a catastrophic event.

Virtualization & Hypervisor

The abstraction of physical hardware, allowing multiple Virtual Machines (VMs) to run on a single physical host, enabled by a hypervisor (Type 1 or Type 2).

Multi-Tenancy

Multiple customers share the same infrastructure resources, but their data and workloads are logically isolated from each other for security.

CAP Theorem

For any distributed data store, you can only have two of three guarantees: Consistency, Availability, and Partition tolerance.

Performance Metrics

Latency: Time delay in data transfer.
Throughput: Rate of data transfer.
Consistency: Ensuring all reads receive the most recent write.
Durability: Ensuring data survives once stored.

Cloud Economics

Cloud converts CapEx (upfront hardware costs) to OpEx (pay-as-you-go). TCO (Total Cost of Ownership) includes all direct and indirect costs.

Service & Deployment Models

IaaS (Infrastructure)

Provider rents you IT infrastructure like servers and storage.

YOU MANAGE: Apps, Data, Runtime, OS

PaaS (Platform)

Provides an on-demand environment for developing and deploying apps.

YOU MANAGE: Applications, Data

SaaS (Software)

Provides ready-to-use software over the internet on a subscription basis.

YOU MANAGE: Your data within the app

FaaS / Serverless

Run event-driven code without provisioning servers.

YOU MANAGE: Code, Logic, Event Triggers

CaaS (Containers)

Managed container orchestration platform (e.g., Kubernetes).

YOU MANAGE: Container Images, Configuration

DBaaS (Database)

Managed database with automated patching, backups, and replication.

YOU MANAGE: Schema, Queries, Data

Public Cloud

Shared resources. Ex: AWS, Azure, GCP.

Use Case: Startups, global web apps.

Private Cloud

Used exclusively by one organization.

Use Case: Banks, healthcare, government.

Hybrid Cloud

Combines public and private clouds.

Use Case: Legacy app migration, bursting.

Multi-Cloud

Using services from more than one public cloud.

Use Case: Leveraging best services, avoiding vendor lock-in.

Cloud Architecture & Networking

Global Infrastructure

Region: Geographic location.
AZ: Isolated data center in a region. Deploy across multiple AZs for HA.

VPC / VNet

A logically isolated virtual network. Contains subnets, route tables, and NAT gateways to manage traffic flow.

Load Balancers

Distributes traffic across targets. Layer 4 (Network) for TCP/UDP traffic. Layer 7 (Application) for HTTP/HTTPS.

Connectivity

VPN/Direct Connect/ExpressRoute: Securely connect on-premises data centers to the cloud for hybrid architectures.

CDN & Edge Computing

A Content Delivery Network caches static content at edge locations closer to users, reducing latency for global applications.

Well-Architected Framework

Pillars: Operational Excellence, Security, Reliability, Performance Efficiency, Cost Optimization. Principles include designing for failure and loose coupling.

Serverless & Containers

Serverless (FaaS)

Pros: No server management, pay-per-execution model.
Cons: Cold starts, statelessness, vendor limitations.

Containers & Microservices

Containers package code and dependencies into immutable images. This enables a microservices architecture with separation of concerns.

Kubernetes Orchestration

Manages container lifecycles. Key concepts: Clusters, Control Plane vs Data Plane, Pods, Services, Ingress Controllers, and Service Meshes (e.g., Istio).

Security & Compliance

Shared Responsibility Model

Provider: Security OF the Cloud (hardware).
Customer: Security IN the Cloud (data, access, configs).

Identity & Access (IAM)

Manages user identities and permissions (who can do what). Follow the principle of least privilege using Roles, Policies, and RBAC.

Encryption & Secrets

Protecting data at rest and in transit. Use Key Management Services (KMS) and secrets management tools (e.g., Vault, Key Vault).

Network Security

Use Firewalls, WAFs, Security Groups (stateful), and Network ACLs (stateless) to control traffic flow and secure endpoints.

Zero Trust Architecture

A security model that never trusts by default. Always verify identity, enforce least privilege, and continuously monitor every access request.

Threat Detection

Utilize services like AWS GuardDuty, Azure Sentinel, or GCP Security Command Center for intelligent threat monitoring and alerts.

DevOps & Automation

Infrastructure as Code (IaC)

Managing infrastructure with code (e.g., Terraform, CloudFormation), enabling version control, repeatability, and automation.

CI/CD Pipelines

Automating build, test, and deploy stages. Use strategies like Blue/Green deployments or Canary releases to minimize risk.

Monitoring & Observability

Collecting metrics, logs, and traces to understand system health. Key for debugging distributed systems (e.g., with AWS X-Ray).

Cost Optimization & FinOps

Right-Sizing

Choose correct VM sizes and storage types.

Reserved/Savings Plans

Commit to usage for a large discount.

Spot/Preemptible Instances

Use spare capacity for non-critical workloads.

Storage Tiering

Move data to cheaper, colder storage tiers.

Shut Down Idle Resources

Automate shutdown of non-prod environments.

Data Transfer Costs

Monitoring & Alerts

Use budgets and tagging to track spending.

FinOps

Cultural practice of financial accountability for cloud spend.

Comprehensive Interview Q&A

Trade-offs between vertical and horizontal scaling?

Vertical (scaling up): Easier, but has a physical limit and creates a single point of failure. Can require downtime.

Horizontal (scaling out): Better fault tolerance and scalability, but introduces complexity (load balancing, data consistency).

How does Shared Responsibility Model vary for IaaS, PaaS, SaaS?

IaaS: Customer manages OS, middleware, runtime, data, app code.
PaaS: Customer manages only the app code and data.
SaaS: Customer manages only their data within the app and user access.

What is a CDN and how does it improve performance?

A CDN caches static content at edge locations close to users. This reduces latency by serving content from a nearby server instead of the origin, improving performance for global users.

Explain "serverless cold start" and how to mitigate it.

A "cold start" is the latency when a serverless function is invoked after being idle. Mitigations include Provisioned Concurrency (keeping instances warm), periodic "pinging", and optimizing code size.

Difference between eventual and strong consistency?

Strong Consistency: Every read receives the most recent write. Essential for financial transactions or inventory systems.
Eventual Consistency: After a write, reads will "eventually" see the update. Acceptable for social media feeds, likes, where a small delay is not critical.

What is Infrastructure as Code (IaC) and why is it important?

IaC is managing infrastructure with code (e.g., Terraform, ARM). It's important because it enables version control, repeatable deployments, automation, and prevents "infrastructure drift" where configurations become inconsistent over time.

Real-World Scenarios

Scenario: Rapid-Growth Social Media App

  • Deployment: Public Cloud across multiple regions.
  • Compute: Horizontal scaling with Auto Scaling Groups and Load Balancers.
  • Database: NoSQL (e.g., DynamoDB) for flexible schema, possibly with a relational DB for transactions.
  • Content: CDN (e.g., CloudFront) for low latency media delivery.
  • Event-driven tasks: Serverless (e.g., Lambda) for image processing.

Scenario: Bank Modernization with Regulations

  • Model: Hybrid Cloud.
  • Data: Sensitive data remains in a private, on-premises data center.
  • Applications: New customer-facing apps are built on the Public Cloud.
  • Connectivity: Secure, private connection (e.g., AWS Direct Connect, Azure ExpressRoute).
  • Security: Strong IAM, encryption everywhere, and robust audit trails.
