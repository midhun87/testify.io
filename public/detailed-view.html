Core Concepts & User Roles

Admin

The superuser with complete control over the entire platform. Admins manage content, users, and system-wide settings.

Moderator

A role to help manage the platform with a subset of admin privileges, typically limited to assigned colleges.

Student

The end-user of the platform. Students engage with learning content, take assessments, and track their progress.

Admin & Moderator: Platform Management

AI-Powered Content Creation

To save educators countless hours. Admins can upload a PDF, and Google's Gemini AI automatically structures it into a digital test or a full course.

User & Access Control

To ensure the platform is secure and users have the correct permissions. Admins and Moderators can manage accounts within their scope.

Comprehensive Reporting & Analytics

To provide deep insights into student performance via dashboards, detailed reports, and contest leaderboards.

Student Experience: Learning & Assessment

Personalized Dashboard

To give students a central hub for their learning journey, summarizing performance, new tests, and recent activity.

Secure & Proctored Exams

To maintain academic integrity. The system uses AWS Rekognition to verify a student's identity via webcam against their profile photo.

Achievements & Verification

To reward effort with verifiable credentials. The system can auto-issue digital certificates with a unique, publicly verifiable ID.

CodeLab & Contests: Practical Skills

Live Code Execution

Why?

To provide an integrated environment for practicing code. The platform uses the OneCompiler API to run code in multiple languages and execute SQL queries.

Coding Contests

Why?

To assess practical coding skills in a competitive environment. The system automatically evaluates submissions against hidden test cases and generates a leaderboard.

Careers Portal: Connecting Talent

Job Board Management

Why?

To allow the organization to easily advertise open positions on a public-facing careers page.

Detailed Application Process

Why?

To streamline hiring by collecting candidate information and documents in a structured way.

Secure Application Tracking

Why?

To allow non-registered users to check their application status securely via an OTP sent to their email.
